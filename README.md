# LLM Latency Tracker

Daily benchmarks of LLM API response times — Time To First Token (TTFT), p50 and p95 latency.

**Updated automatically every day.**

## Metrics

- **ttft_ms** — Time To First Token (streaming, ms from request to first chunk)
- **total_ms** — Total time to complete response
- **p50_ms / p95_ms** — Percentile latencies across N=5 runs per model
- **tokens_per_second** — Throughput estimate (output tokens / total_ms * 1000)

## Usage

This is a public data feed. Use it to:
- Track LLM provider SLA trends
- Detect regional latency spikes
- Build dashboards or alerts

## Data files

- `data/YYYY-MM-DD.json` — daily benchmark run
- `data/latest.json` — always points to most recent run
- `data/summary.json` — rolling 30-day aggregates
